# CbD Assessment and Data — Operational Reference
*Added: 2026-03-29 | Full research: `Research/CbD_Assessment_Data_Framework.md`*

**Load this file when:** building any rubric, IEP goal stem, session tracker, or data collection tool for any CbD product. Also load when reviewing a unit for IDEA progress monitoring compliance.

---

## The Four CbD Data Types

| Type | Used For | Tool |
|------|----------|------|
| **Rubric scoring** | Response activities (Perspective Comparison, CER, ESR, Source Analysis) | 3-level rubric on the student's response document |
| **Event recording** | AAC symbol output; annotation accuracy; vocabulary identification | Tally on teacher/para form or Session Tracker |
| **Permanent product** | Completed graphic organizers, CER frames, annotation guides | The student's work sample IS the data |
| **Session Tracker** | Per-session AAC communication data | CAP Communication Session Tracker |

---

## Rubric Design Rules (Non-Negotiable)

- 3 levels: **Does Not Yet Meet / Approaching / Meets**
- Each level = observable student behavior (not "excellent" / "good" / "needs work")
- "Meets" criterion = the grade-level standard (not lowered for V2/V3)
- Rubric is **identical across V1/V2/V3** — version changes the access; rubric measures the standard
- Rubric is **on the student's response document**, not a separate sheet
- Rubric must be scorable by a para with no content expertise

**V3 scoring note:** A V3 student who selects a correct option in choice/cloze format scores at "Meets" on the same rubric as a V1 student who writes a complete sentence — both demonstrated the standard. The response mode is different; the expectation is not.

## Default Mastery Criteria by Skill Type

| Skill | Academic Criterion | AAC Criterion |
|-------|-------------------|---------------|
| Close Reading / Annotation | 80%, 3 consecutive sessions | Symbol output 4/5 opp., 2+ sessions |
| Author's Purpose / Perspective | 2 perspectives w/ evidence, 3 of 4 trials | 2+ symbol sequence, 4/5 opp. |
| CER | Claim + 2 evidence aligned, 4 of 5 trials | CER via AAC, 3/5 trials |
| Text Structure | 3 of 4 elements labeled, 3 consecutive | Cause/effect via AAC, 4/5 opp. |
| Sourcing / Corroboration | 3/4 sources rated w/ justification, 2 consecutive | Source rated via AAC w/ justification, 3/5 |

**Minimum mastery criterion:** 70%. Do not lower below this.

**Generalization requirement:** Data across 2+ partners before reporting mastery. One partner = trend, not mastery.

---

## Data Roles

| Role | Collects |
|------|----------|
| Special educator | Rubric scores; session accuracy counts |
| Paraprofessional | Communication Session Tracker; symbol output tallies |
| SLP | Vocabulary pre-programming confirmation; CAP delivery date |
| Student | Self-assessment (when possible) |

**Design rule:** Data collection must be built into the activity. If it requires a separate sheet, it won't happen. Every rubric-scored activity in a CbD unit includes the rubric on the student's response document.

---

## Progress Note Minimum (IDEA §300.320)

1. **Current performance:** "[Student] is scoring X of 5 on [skill] activities using [V#] materials with [support]"
2. **Trend:** "This represents [increase / maintenance / decline] from [baseline] at unit start"
3. **Next step:** "Team will [continue / advance version / adjust criterion] based on this data"

**Not a progress note:** "Student enjoyed the lesson." / "Student is making progress." / "Student participated in activities."

---

## QC — Pre-Build Data Checklist

- [ ] Rubric included for every scored response activity?
- [ ] Rubric is on the student response document (not separate)?
- [ ] Rubric uses behavioral descriptions at each level?
- [ ] Rubric is identical across V1/V2/V3?
- [ ] Session Tracker in CAP?
- [ ] IEP goal stems include performance criterion + consistency criterion?
- [ ] Generalization across 2+ partners referenced?
- [ ] Default mastery criterion ≥70%?
- [ ] Data role assignments in teacher guidance?
