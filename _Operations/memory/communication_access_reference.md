# CbD Communication Access — Operational Reference
*Added: 2026-03-29 | Full research: `Research/CbD_Communication_Access_Framework.md`*

**Load this file when:** designing any product activity, reviewing a unit for AAC compliance, building a new product line, assigning vocabulary to the CAP, or writing partner guidance.

---

## What Communication Access Means

Communication Access = systematic embedding of AAC-compatible design so that any student who uses **any form of AAC** (SGD, eye-gaze board, symbol cards, e-trans board, alternative pencil) can participate fully in the instructional activity and demonstrate understanding through their own communication system.

**It is NOT:**
- A simplified version of the lesson
- A V3-only feature
- The SLP's job alone
- An add-on after the unit is built

**It IS:**
- A design standard embedded in every activity at the point of authorship
- Active at all version levels (V1, V2, V3)
- A team responsibility (special educator + para + SLP + classroom teacher)

---

## The CbD AAC Hierarchy

AT → AAC → SGD

CbD designs for **AAC** — which includes SGDs and all low-tech systems. Never assume SGD only.

---

## Universal Design Rules (All Product Lines)

1. **Vocabulary first.** No product goes to build without a confirmed vocabulary list (core + fringe).
2. **Partner guidance at point of use.** Not in an appendix — embedded in the activity.
3. **No oral-only response opportunities.** Every activity must have a non-speech pathway.
4. **Response mode flexibility.** Accept: AAC output, gesture, eye gaze, pointing, writing, circling.
5. **Speakable everything.** Every prompt, frame, and option readable aloud by a partner.
6. **Consistent core vocabulary** across all product lines: *because, show, prove, agree, different, same, not, true, wrong* → builds one motor pathway that transfers.

## The Communication Access Ecosystem

| Product Line | Access Layer | Response Target | CAP Role |
|-------------|-------------|----------------|----------|
| **UFLI Phonics (K-2)** | Communication Partner Guide per lesson | Symbol output for word ID tasks | Partner Guide + Session Tracker equivalent |
| **Chapter Book / Fiction (Gr 3-8+)** | Visual Scene Displays + story grammar vocabulary | 2–3 symbol utterances; story grammar | CAP delivers emotional + story grammar fringe |
| **Nonfiction (Gr 6-10)** | Full CAP + V3 vocabulary instruction | Multi-symbol CER/analysis responses | CAP delivers content fringe; SLP confirms pre-programming |
| **Picture Book Companions (K-3)** | 3-reading structure (IRA + Dialogic Reading) | Partner-prompted → spontaneous 1-2 symbol | Vocabulary list (15-20 words max) |

**The progression goal:** Students who build communication routines in UFLI + fiction arrive at nonfiction ready to use their AAC system for analysis — not still learning the motor pathway.

**Chapter book line (in development):** Shared reading structure with designated AAC response points per chapter. Designed to develop language access before secondary nonfiction complexity demands.

## AAC Access Across V1/V2/V3

| Version | Vocabulary | Partner Role |
|---------|-----------|-------------|
| V1 | Pre-programmed via CAP | Models AAC during instruction; device/board available |
| V2 | Pre-programmed via CAP | Models fringe; sentence frames support motor planning |
| V3 | Pre-programmed via CAP **+ vocabulary TAUGHT in unit** | Reads aloud; teaches vocabulary step; models output; 5-sec wait |

---

## The CAP — What It Delivers

| Recipient | What They Receive |
|-----------|------------------|
| SLP | Fringe vocabulary list + Fitzgerald categories + ARASAAC symbol refs. Due 1-2 weeks before Day 1. |
| Para | Session Tracker + modeling vocabulary guidance |
| Special educator | Vocabulary confirmation checklist + Session Tracker summary |
| Classroom teacher | Partner guidance embedded in teacher document (not a separate CAP) |

**CAP is non-optional for V3.** Without pre-programmed vocabulary, the V3 vocabulary instruction step cannot happen.

---

## Progression Indicators

| Product Line | Start Target | End Target |
|-------------|-------------|-----------|
| UFLI Phonics | 1 symbol/response, partner-prompted | Consistent symbol output for word ID tasks |
| Picture Book | 1 symbol, partner-prompted | 1–2 symbols with reduced prompting |
| Fiction | 1–2 symbols, partner-prompted | 2–3 symbol utterances; story grammar vocabulary |
| Nonfiction | Pre-programmed fringe, partner-prompted | Fringe across partners; multi-symbol CER responses |

**Progress ≠ moving to a higher version.** Communication progression and academic progression are separate. The IEP should include goals for both.

## QC — Communication Access Pre-Build Checklist

- [ ] Core vocabulary set confirmed (consistent with cross-unit motor pathway)?
- [ ] Fringe vocabulary list complete?
- [ ] CAP timeline documented (1–2 weeks before Day 1)?
- [ ] Every activity in every version has at least one AAC response opportunity?
- [ ] No activity accepts oral response only?
- [ ] All prompts written to be read aloud?
- [ ] Partner guidance embedded at point of use (not in appendix)?
- [ ] Partner guidance names vocabulary to model and when?
- [ ] 5-second wait time named in seconds in partner guidance?
- [ ] Session Tracker included and assigned to para?
- [ ] Data collection distinguishes academic progress from communication progress?
