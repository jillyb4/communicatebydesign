# CbD IEP Goal Ecosystem — Operational Reference
*Added: 2026-03-29 | Full research: `Research/CbD_IEP_Goal_Ecosystem.md`*

**Load this file when:** writing IEP goal stems for a new unit, reviewing goal stems for IDEA compliance, or supporting a teacher in connecting a CbD unit to an existing IEP.

---

## The Key Distinction

**CbD products are not IEP goals. They are the instructional vehicle.**

The goal stem included in every CbD unit is a *starting point* for teams — not a required form, not a compliance guarantee, not a substitute for professional judgment. Teams write the actual goal. The stem shows what IDEA-compliant language looks like for this skill.

---

## IDEA Minimum Requirements for a Measurable Annual Goal

Every IEP goal connected to a CbD unit must include all five elements:

```
Given [condition: version + specific supports],
[student] will [observable behavior verb] [skill + content standard]
as measured by [data tool that exists in the product],
achieving [performance criterion] + [consistency criterion] by [date].
```

**Accommodation test (run before finalizing):**
> "Is the ELA standard in this goal the same standard peers are working toward, with a different access method?"
- Yes → accommodation-based goal (correct for most CbD users)
- No → modification — requires documentation and qualifies only for 1% alternate achievement standard population

## Observable Verbs by Skill Type

| Skill | Use These Verbs |
|-------|----------------|
| Close Reading / Annotation | identify, annotate, cite, locate, mark |
| Author's Purpose / Perspective | explain, distinguish, compare, defend, support with evidence |
| CER | state, identify, select, write, construct |
| Text Structure | label, organize, sequence, categorize |
| Sourcing / Corroboration | evaluate, rate, compare, justify |
| AAC communication output | indicate, select, produce, use [specify modality] |

**Never use:** understand, know, appreciate, be aware of, learn — not observable; fail IDEA measurability.

## The Two-Goal Model (Academic + AAC)

For students using AAC in CbD units, write two separate goals:

| Goal | Measures | Data Tool |
|------|---------|-----------|
| Academic ELA | Did the student demonstrate the ELA skill? | CbD rubric on student response document |
| AAC communication | Did communication output develop in context? | Communication Session Tracker |

Academic progress and communication progress are not the same. Both need separate goals and separate data.

## Default Mastery Criteria (from Assessment & Data Framework)

| Skill | Academic Criterion | AAC Criterion |
|-------|-------------------|---------------|
| Close Reading / Annotation | 80%, 3 consecutive sessions | 4/5 opp., 2+ sessions |
| Author's Purpose / Perspective | 2 perspectives w/ evidence, 3 of 4 trials | 2+ symbol sequence, 4/5 opp. |
| CER | Claim + 2 evidence aligned, 4 of 5 trials | CER via AAC, 3/5 trials |
| Text Structure | 3 of 4 elements labeled, 3 consecutive | Cause/effect via AAC, 4/5 opp. |
| Sourcing / Corroboration | 3/4 sources rated w/ justification, 2 consecutive | Source rated via AAC, 3/5 |

Minimum mastery criterion: 70%. Generalization: across 2+ partners before reporting mastery.

## Common Errors

| Error | Fix |
|-------|-----|
| "understand" / "know" in the goal | Replace with observable verb from the table |
| No performance criterion | Add: "4 of 5 trials," "80% accuracy" |
| No consistency criterion | Add: "across 3 consecutive sessions" |
| No data method | Name the tool: "as measured by rubric scoring on CER activities" |
| Academic + AAC collapsed into one goal | Split into two goals with separate data tools |
| Goal stem copied verbatim | Adjust condition (version + supports), criterion (from baseline), date |

---

## Build Rule for Goal Stems in New Products

Every CbD unit ships with at minimum:
- One academic goal stem (for the unit's skill type)
- One AAC communication goal stem (cross-unit format)

Both must follow the full IDEA structure. Cross-reference the Assessment and Data Framework for default criteria before finalizing.
