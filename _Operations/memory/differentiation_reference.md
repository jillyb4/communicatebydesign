# CbD Differentiation — Operational Reference
*Updated: 2026-03-29 (Version 2.0) | Full research: `Research/CbD_Differentiation_Framework.md`*

**Load this file when:** designing V1/V2/V3 text versions, writing activity scaffolds, assigning HLPs to activity types, or reviewing a unit for IDEA legal defensibility.

---

## The IDEA Compliance Test (Run First — Non-Negotiable)

**Every V3 activity must pass before it ships:**

> "Can a student who completes this activity be said to have *demonstrated the skill* the standard requires? Or has the task been reduced to factual recall?"

- Demonstrates the skill → **Accommodation** — permitted for all students with IEPs
- Reduced to recall only → **Modification** — only permitted for the 1% alternate achievement standard population

CbD's V1/V2/V3 model is an **accommodation framework**. All three versions target the same standard.

---

## Version Definitions (Authoritative)

| Version | Reading Level | Design Intent | Key Feature |
|---------|--------------|---------------|-------------|
| **V1** | Grade-level (nonfiction: 900–1050L) | Full rigor; same standard; open-ended response | Grade-band Lexile; no structural adaptation |
| **V2** | 5th/6th grade comprehension (nonfiction: 650–800L) | Same skill, same activities, same HLPs — reading barrier removed; one focus skill | Sentence frames; text restructured |
| **V3** | 3rd/4th grade reading (nonfiction: 400–550L) | Same skill, same activities, same HLPs — vocabulary TAUGHT in unit + choice/cloze | Descriptive Teaching Model vocabulary instruction + pre-programming; AAC output compatible |

**V2 principle:** V1's complexity comes from the text AND the skill. V2 removes the text complexity so the skill is the only thing the student is working on. Same HLPs. Same activities.

**V3 principle:** Reading level AND vocabulary access are both barriers. V3 removes both. Vocabulary is actively taught (not just listed) using the Descriptive Teaching Model. Same skill. Same HLPs.

---

## AAC Access — ALL Versions (Not V3-Only)

| Version | Vocabulary | Partner Role |
|---------|-----------|-------------|
| V1 | Pre-programmed via CAP | Models AAC while student reads; device/board available |
| V2 | Pre-programmed via CAP | Models fringe; sentence frames support motor planning |
| V3 | Pre-programmed via CAP **+ vocabulary TAUGHT in unit** | Reads aloud; teaches vocabulary step; models output; 5-sec wait |

**Hard rule:** Version assignment is a reading access decision — NOT a communication access decision. A robust SGD user at grade level uses V1 with the CAP active.

## The Locked Constants (Identical Across V1/V2/V3)

| Element | Rule |
|---------|------|
| ELA standard | Identical across all versions |
| Essential question | Identical across all versions |
| Vocabulary list | Same words, same Top 5 core/fringe, same CAP |
| Historical events and evidence | Every fact in V1 must appear in V3 |
| Skill type | Close Reading / Text Structure / CER / Author's Purpose / Sourcing — locked |
| Response task category | If V1 = claim + evidence, V3 = claim + evidence (format scaffolds, task does not) |
| **High-Leverage Practices (HLPs)** | Same HLP category across all versions — the practice doesn't change; the scaffold does |
| **AAC access pathway** | Present and active at all three version levels |

## The Three Differentiation Mechanisms

### 1. Text Complexity (Lexile)

| Version | Lexile | Words/Passage | Design Intent |
|---------|--------|--------------|---------------|
| V1 | 900–1050L | 550–650 | Near-original source language |
| V2 | 650–800L | 450–550 | Restructured sentences; same events; one skill focus |
| V3 | 400–550L | 330–400 | Chunked sentences + active vocabulary instruction |

**Reduction tools (in order):**
1. **Sentence decomposition** — break long multi-clause sentences. Preserve all information.
2. **Syntactic simplification** — passive → active; embedded clauses → separate sentences.
3. **Knowledge demand reduction + Vocabulary Instruction (V3 only)** — Descriptive Teaching Model pre-reading step for 3–4 fringe words.

**Hard rule:** V3 does NOT remove evidence. Every fact present in V1 must appear in V3.

---

### 2. Activity Scaffold Gradient

| Format | Version | Pattern |
|--------|---------|---------|
| Open-ended | V1 | Blank space; student produces full response |
| Partial structure | V2 | Sentence frame; student fills evidence and analysis |
| Choice/cloze access | V3 | Options embedded; student selects AND supplies at least one content element |

**Sentence frames rule:** Present in all versions. Must require student to supply content. Must target the same grammatical structure. Use shared core vocabulary: `because, show, prove, agree, different, same, not, true, wrong`.

---

### 3. AAC Response Pathway (Overlay — All Versions)

- Core response vocabulary on communication board for ALL activities across ALL versions
- Fringe vocabulary pre-programmed via CAP → SLP (1–2 week lead time) for V2 and V3
- V3: vocabulary also TAUGHT within the unit (Descriptive Teaching Model) in addition to pre-programming
- All prompts written to be read aloud — no version assumes silent independent reading
- All response options are valid: AAC output, oral, pointing, writing

## HLP Alignment by Activity Type

| Activity | HLP | What Differentiates V1→V3 |
|----------|-----|--------------------------|
| Close Reading / Annotation | HLP 12 | Scaffold level (teacher model → partner-guided → symbol-coded) |
| CER | HLP 14 | Open frame → partial frame → choice + frame |
| Perspective Comparison | HLP 13 | Response format (open → structured → choice) |
| Source Analysis | HLP 12 + 14 | Source features made explicit; rating scaffold tightened |

## V3 Vocabulary Instruction Format

V3 includes an active teaching step — not a glossary.

```
KEY VOCABULARY — BEFORE YOU READ
[Word] — [Student-facing definition, plain language]
[Word] — [Student-facing definition, plain language]
```

Followed by: partner says the word → student finds it on device/word card → partner uses it in a sentence from the passage. Maximum 3–4 words per passage. All pre-programmed via CAP.

## Common Differentiation Errors

| Error | Rule Violated |
|-------|--------------|
| Removing an event from V3 to shorten the passage | All facts must appear in V3 |
| V3 activity asks for identification only (not analysis) | Response task category is locked |
| Different vocabulary lists across versions | Vocabulary is a locked constant |
| V3 vocabulary box is a glossary, not a teaching step | V3 requires Descriptive Teaching Model instruction |
| V3 omits AAC access because "it's for SGD users only" | AAC access is at every version level |
| HLP label removed from V3 activity | HLPs are locked — practice category does not change |
| Version assigned by disability label | Version = reading access + vocabulary access decision |

## ⚠️ Poetry Line Exception — Access-Layer Model

**Poetry Reading Units do NOT use the Lexile text-versioning model.**

The nonfiction V1/V2/V3 model assumes a prose passage that can be rewritten at different Lexile levels while preserving all factual content. That assumption breaks for poetry: rewriting Paul Laurence Dunbar's "We Wear the Mask" at a 400L reading level produces a different poem. The analytical content — the compression, the line breaks, the sound — is the text. Simplifying it destroys the very thing the standard asks students to analyze.

**Instead, poetry uses the access-layer model:**

| Element | Nonfiction V1/V2/V3 | Poetry V1/V2/V3 |
|---------|---------------------|-----------------|
| Text | Three different Lexile versions | **Same poem across all versions** |
| What changes | Reading level of the passage | Annotation scaffolds, response frames, partner support intensity |
| What stays the same | Standard, skill, vocabulary, facts | Standard, skill, vocabulary, **the poem itself** |
| Analogous line | Fiction / Picture Book Companions | Fiction / Picture Book Companions |

### What V1/V2/V3 Means for Poetry

| Version | Access Layer | Partner Role |
|---------|-------------|--------------|
| **V1** | Poem printed; NFMA frames present; response format open-ended | Models AAC; provides 5-sec wait; no additional scaffolding |
| **V2** | Poem printed with text annotations and guided NFMA prompts embedded; sentence frames with fill-in structure | Models fringe vocabulary; reads prompts aloud; reduces response field |
| **V3** | Poem printed with symbol-coded annotations; vocabulary pre-taught using Descriptive Teaching Model before reading; choice/cloze NFMA frames; stanza-by-stanza pacing | Reads every line aloud; teaches vocabulary step; models AAC output at each NFMA prompt; 5-sec wait named in seconds |

**Locked constants (identical across all three poetry access levels):**
- The poem text (word-for-word identical)
- ELA standard and essential question
- Vocabulary list (same words, same Top 5 core/fringe, same CAP)
- NFMA strategy (all four steps present at every level)
- Response task category (if V1 = analyze figurative language in evidence, V3 = same task with cloze/choice scaffold)
- HLPs (HLP 12 annotation, HLP 14 CER-style response, HLP 15 partner support)
- AAC access pathway (active at all three levels)

### IDEA Compliance Test — Poetry Version

The standard access test still applies to V3 poetry activities:

> "Can a student who completes this activity be said to have *demonstrated the skill* (analyzing figurative language / identifying speaker / analyzing structure) — or has the task been reduced to factual recall?"

**Common poetry V3 compliance failures:**
- Activity asks student to identify what kind of word (*metaphor*) but not what it means → recall only, not analysis
- Activity removes all open response elements and replaces entirely with yes/no → modification
- NFMA step MEAN is cut for V3 because "it's too hard" → standard is removed, not scaffolded

**Hard rule:** V3 must reach the MEAN and ASK steps of NFMA. The scaffold controls HOW the student gets there — it does not remove the destination.

### What Does NOT Change from the Nonfiction Model
- Pre-build vocabulary gates (required before build)
- CAP vocabulary list (SLP handoff, 1–2 week lead time)
- Communication Partner Framework (5-second wait, ALgS modeling, choice offering)
- Session Tracker assignment (para)
- Rubric design (3-level, identical across versions — "Does Not Yet Meet / Approaching / Meets")

## QC Gates (Run Before Finalizing Any Version)

**Gate 1: IDEA Compliance**
- [ ] V3 activity passes the access test (skill demonstrated, not just recalled)?
- [ ] All version differences documentable as accommodations?

**Gate 2: Text Complexity**
- [ ] V1/V2/V3 Lexile confirmed within bands?
- [ ] Word count within target for each version?
- [ ] All V1 facts present in V3?

**Gate 3: Locked Constants**
- [ ] Essential question identical across versions?
- [ ] Vocabulary list identical across versions?
- [ ] Skill type identical across versions?
- [ ] HLPs labeled and identical across versions?
- [ ] AAC access pathway present in all three versions?

**Gate 4: Activity Scaffolds**
- [ ] V3 choice/cloze still requires student to supply content?
- [ ] Sentence frames present in all versions and target same structure?
- [ ] Response task category identical across versions?

**Gate 5: V3 Vocabulary Instruction**
- [ ] Key Vocabulary section uses Descriptive Teaching Model format?
- [ ] Teaching step (not just definition) included?
- [ ] Words pre-programmed via CAP?
