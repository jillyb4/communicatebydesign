Phase 0 · Topic Analysis
0a
Read Master Reference
Always first. Grounds you in the design system, skill grid, and existing product inventory.

CbD_Unit_Design_Master_Reference.md
0b
Research the Topic
Primary sources, historical accounts, photos, docs, first-person narratives. Ask: How rich is the material? Whose voices are present / missing? Is there existing curriculum coverage?
0c
Age Range & Version Strategy
Let the content decide the audience. Check for age-sensitive themes. Lock V1/V2/V3: same skill, same expectation, different access.

V1: 900–1050 · V2: 650–800 · V3: 400–550
0d
Arc & Scope
Decide passage count (1–4): 1 passage = mini-lesson/freebie · 2 passages = comparison/shift (Capitol Crawl model) · 3–4 passages = full narrative arc (504 model)
0e
Select Target Skill
Choose from skill grid. Skill shapes the text interaction tool, graphic organizer, and all comprehension activity design. Confirm skill number + tool + organizer type.
0f
Write Essential Question
Arguable, not googleable. Connects to justice, identity, power, responsibility, or voice. Appears on title page + teacher materials. Drives discussion + project work.
0g
Evaluate Project Potential
Does the topic support synthesis, real-world action, or position-taking? If yes → design a project component. If no → comprehension activities are enough.
0h
✓ Checkpoint: User Approves
Confirm before ANY drafting: essential question · age range · V1/V2/V3 strategy · passage count · arc · target skill + tool + organizer · project or no project.

Phase 3 · Build
A
2-Part Lesson Script (Capitol Crawl / Zitkala-Ša model)
~960 lines · Build script in product folder · require template via relative path to _Operations/Build/cbd_docx_template.js · Passage extraction via regex · Output: {Name}_Lesson_COMPLETE.docx
B
4-Part Unit Script (504 Sit-In / Frances Kelsey model)
~1,500–2,000+ lines · Build script in product folder · require template via _Operations/Build/cbd_docx_template.js · Multi-session work · Build in phases: teacher docs → passages → comprehension → answer keys · Rebuild + validate XML after each phase
⚙ Critical Build Warnings — Never Skip
⚠ Always spread array-returning template functions (missing ... causes <0/> XML corruption)
⚠ studentHandoutHeader takes exactly 3 args: (unitTitle, title, subtitle)
⚠ V3 passage: strip **Key Vocabulary** heading before passing to buildPassageParagraphs (avoids duplicate header)
⚠ V3 regex boundary: use \*Word count: or ## as end boundary — NOT ---\s*\n
⚠ MCQ spacing: use before: 160 (not 280) for question stems to keep all 5 on one page
⚠ Table headers: no special chars (¶, §) — they render literally in Word. Plain text only.
Standard Quick Wins — Apply to Every Unit
Word Parts morphology handout (3–5 Tier 2/3 words)
Student-facing learning targets on every primary handout
Text structure / skill-specific visual reference card (half-page)
Scaffold fading rationale (teacher-facing, cite HLP 15)
V3 DOK 3 MCQ: at least 1 structure/skill-analysis question

Structural
Zero XML corruption (<0/> tags)

Content
All passages present (versions × parts)
All comprehension activities present (by version)
Answer key complete
Accessibility Statement included
Essential question appears in multiple locations
22 standard sections present (nonfiction units)
Inclusive Language
Zero deficit-framing (no "below grade level," "struggling readers")
"point to" always paired with inclusive alternatives
AAC-inclusive verbs: select, indicate, identify
Strengths-based language throughout
"Communication Access" (not "AAC Support") as section heading
PDF: Word → Save As PDF (not Print to PDF, not LibreOffice)
⚠ Goals & Data
IEP goal stems target language growth — not compliance, not device operation
Data tools measure language growth indicators — not accuracy/prompt level/trial count alone
Observable IDEA verb: Given [condition] → will [verb] → measured by [tool] → [criterion]
Goals framed as accommodations — same standard, different access
Two-goal model: academic ELA goal + AAC communication goal per unit

Non-Negotiable Rules
Hard Rules — Never Violate
Never call Jill "AT Specialist" or "AAC Specialist" — not yet certified. Title: "special educator and advocate"
Never "above/below grade level," "struggling readers," "low students" — use V1/V2/V3 Lexile framing
Never "point to" as the sole interaction verb — "select, point to, or use gaze to indicate"
Never default to "work with the SLP" — frame as IEP team responsibility
Never assume the student has an SGD — must work with alternative pencil + e-trans board + symbol cards
Never add words to a UFLI lesson — the word list is fixed and research-based
Never abbreviate "CbD" in outward-facing content — always "Communicate by Design"
Never change existing unit pricing — use the tiered price table
Every nonfiction unit requires 22 standard sections → nonfiction_build_reference.md
AAC core/fringe word TABLE required in every unit's Differentiating section — makeTable(), not bullet lists
Every Output 0 must be in the listing package BEFORE any other section
Never rebuild something already approved — str_replace over full rebuilds always
